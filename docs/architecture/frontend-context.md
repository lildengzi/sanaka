# Frontend Context

## 目的

这份文档只解释 **当前前端界面里“已经画出来的内容是在做什么”**，不讨论后端实现，也不假设它已经真的能驱动 QEMU。

当前前端主要是一个：

- 空状态主页
- “关于 Sanaka” 弹出卡片
- “新建虚拟机” 右侧抽屉表单

它表达的是一个 **QEMU 虚拟机管理器的创建流程原型**。

## 主页部分

### 顶部栏

顶部栏现在只有两个作用：

- 左侧显示应用名 `Sanaka`
- 右侧三点按钮打开“关于 Sanaka”卡片

这部分不是虚拟机操作区，更像应用级导航的占位。

### 空状态

主页中间的空状态在表达：

- 当前还没有虚拟机
- 用户应该点击右下角按钮创建第一个虚拟机

它的作用是首页占位，而不是虚拟机列表页。

### 右下角 FAB 按钮

右下角圆形 `+` 按钮的意思是：

- 打开“新建虚拟机”抽屉

这是当前页面最主要的主操作入口。

## 关于卡片

“关于 Sanaka”卡片包含：

- Logo
- 产品名
- 一句描述：轻量 QEMU 管理界面
- 版本和署名信息

它只是信息展示，不参与虚拟机配置。

## 新建虚拟机抽屉

右侧抽屉是当前前端最核心的内容。它想表达的是“创建一台虚拟机时可配置的参数”。

当前抽屉里的区块如下。

### 1. 信息

这块是虚拟机的基础元信息：

- `虚拟机名称`
- `注释`

它们更偏用户侧管理信息，不是 QEMU 硬件参数。

### 2. 基础配置

这块是在表达“启动一台虚拟机最核心的机器参数”。

包含：

- `操作系统（模板）`
  - 当前更像预设入口
  - 例如 Windows 98 / Windows 10/11 / Linux
  - 还带一个系统提示文案
- `QEMU 架构`
  - 例如 `x86_64`、`arm`、`aarch64`
- `第一启动方式`
  - 默认 / 光盘 / 硬盘 / 软盘
- `光盘镜像`
  - 表示安装介质或启动镜像路径
- `声卡`
  - 选择模拟设备型号
- `虚拟机加速设置`
  - `tcg` / `kvm` / `hvf` 等

这一组本质上是在模拟“新建虚拟机向导”的机器基础页。

### 3. 磁盘配置

这块表达的是：

- 给虚拟机挂载硬盘镜像
- 最多允许配置 4 个硬盘
- 支持从本机挑选已有磁盘文件

当前前端里真正有一部分交互：

- 点击“添加磁盘”会通过 Electron IPC 打开系统文件选择器
- 选中的文件路径会被加入前端内存列表
- 可以在列表里删除

但要注意：

- 这只是前端状态
- 没有保存到配置文件
- 也没有真的创建磁盘镜像

### 4. 网络

这块是在表达虚拟机网络设置：

- 是否启用网络
- 网络模式
  - `User`
  - `Bridge`
- 网卡类型
  - 各种 QEMU 支持的虚拟网卡型号

意思很明确：让用户决定虚拟机联网方式和模拟哪种网卡。

但当前主要还是展示型表单，还没看到完整的联动逻辑。

### 5. 显示

这块是在表达图形输出相关配置：

- `显卡选择`
  - 例如 `virtio-vga`
- `显示前端`
  - `Spice`
  - `VNC`
  - `Sanaka (Beta)`
- 条件附加项
  - Spice 端口
  - VNC 端口
  - VNC 密码

这部分设计意图是：

- 先选图形设备模型
- 再选最终怎么连显示输出

它是在为以后“如何显示虚拟机画面”留位置。

### 6. 外部设备

这块目前非常轻，只放了：

- `启用 USB-tablet 鼠标`

这说明作者原本想往“外设模拟”扩展，但现在还没展开。

### 7. 高级选项

这里只有一个文本框：

- `自定义 QEMU 参数`

这块的意思很直接：

- 给高级用户留一个手工追加参数的逃生口

也就是即使图形界面没覆盖全部配置，也允许用户自己补命令行参数。

## 当前已有的前端交互

真正已经写出来的主要交互有：

- 点击右上角按钮展开/关闭 about 卡片
- 点击右下角按钮打开/关闭新建虚拟机抽屉
- 点击背景或按 `Esc` 关闭弹层
- 自定义下拉菜单的展开、选择、键盘操作
- 根据系统模板切换系统图标
- 通过系统文件选择器添加磁盘
- 在磁盘列表里删除项目

## 当前只是“看起来有”的内容

很多字段现在只是被画出来，并不代表已经打通业务：

- 虚拟机名称和注释没有提交保存
- ISO 选择器没有完整接线
- 网络配置没有真正影响任何运行参数
- 显示配置没有接实际显示后端
- 高级参数没有进入命令生成
- 页面里没有“创建虚拟机”最终提交动作

所以这份前端更像：

- 交互草图
- 配置面板原型
- 参数收集界面的雏形

而不是完整可用的创建流程。

## 现在这套前端在产品上想表达什么

如果只看意图，不看实现，这套前端想做的是：

1. 首页先显示“没有虚拟机”
2. 用户点击 `+`
3. 右侧打开一个完整参数抽屉
4. 用户把虚拟机名称、系统模板、磁盘、网络、显示等配好
5. 最后应该创建出一台虚拟机

只是第 5 步现在还没有真正落地。

## 简单判断

当前前端的内容方向并不离谱，表达的是一个很标准的“虚拟机创建面板”。

它的问题不在“内容完全错了”，而在：

- 内容太多，第一次就堆满
- 参数层级没有收敛
- 真正已实现的交互很少
- 看上去像完整产品，实际上还是原型

所以理解这套前端时，最好把它当成：

- “Sanaka 的创建虚拟机界面草稿”

而不是：

- “已经做完的虚拟机管理器前端”
