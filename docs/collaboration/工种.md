# Sanaka 工种分工

## 目的

这份文档用于明确 `Sanaka` 项目中的默认协作分工，避免前后端职责混乱，减少重复决策。

这里的分工不是技术能力限制，而是默认主责划分。

## 默认分工

### Gemini / 其他AI

`Gemini / 其他AI` 默认负责前端相关工作，包括：

- UI 设计落地
- 视觉规范执行
- 页面结构与交互整理
- Electron 渲染层界面实现
- React 组件与样式系统
- 前端文案收敛与用户界面表达
- 前端体验优化，如留白、层级、动画、控件统一性

前端范围内，`Gemini / 其他AI` 应优先关注：

- 页面是否清晰
- 用户是否容易理解
- 交互是否稳定
- 视觉是否统一
- 是否遵守 `context.md`、`design-spec.md` 与其他设计文档

### GPT

`GPT` 默认负责后端和基础能力相关工作，包括：

- Electron 主进程能力
- `preload` 桥接与 IPC 接口设计
- 机器包读写
- `.saka` / `machine.svm` 相关文件规则
- 默认存储路径、最近项目、删除、导入导出
- 运行时架构预留
- 配置模型、状态管理、数据流整理
- 测试、构建、工程结构、稳定性修复

后端范围内，`GPT` 应优先关注：

- 数据结构是否稳定
- 接口语义是否清楚
- 文件行为是否符合规则
- 平台行为是否一致
- 前端是否拿到清晰、可用、可维护的能力边界

## 使用方式

后续在 `Sanaka` 项目里讨论方案、写规范、改代码时，默认参考这份文档。

如果未来工种调整，应直接更新本文件，并同步修订 `context.md` 中的引用说明。

## 当前对接方式

- `GPT` 负责后端、运行时、IPC、文件规则和稳定性修复。
- `Gemini` 负责前端、页面结构、视觉、交互和控制台表现。
- 后端需求交给 `gpt-want.md`。
- 前端需求交给 `geminilog.md` 或对应前端交接文档。
- 两边先分别做自己那一层，不要互相改对方主职责里的实现。
