# Kimi 这次把导出功能搞偏的记录

这次要直接记清楚：问题不只是“丑”，而是 Kimi 在没看清现有代码和接口的情况下就开始写，结果把前后端对接方向整个带偏了。

说难听点，这次不是在做功能，是在编功能。

## Kimi 这次具体干了什么离谱事

1. 后端明明已经有导出 API，Kimi 还能写出一份“需要后端新增导出 API”的文档。
   这不是正常遗漏，这是连现有接口都没先核对就开始输出。

2. 真实接口明明是：
   - `window.electronAPI.machine.exportMachine`
   - `window.electronAPI.machine.cancelExport`
   - `window.electronAPI.machine.onExportProgress`

   Kimi 还能凭空写成：
   - `export.exportMachine`
   - `export.onProgress`

   这不是实现偏一点，这是接口名都现编。

3. `files.openFolder` 明明已经接好了，Kimi 还写“需要后端实现”。
   说明它根本没把现有 preload、types、main 端链路过一遍。

4. 导出弹窗里还在用 `setInterval` 假跑进度，居然还能写“前端已完成全部 UI 和交互”。
   这话写得非常不负责。假进度不是交互完成，顶多算演示壳子。

5. 目录选择居然拿 `pickDisk()` 冒充“选文件夹”。
   这个不是审美问题，是功能逻辑都没站稳。

## 本质问题

这次 Kimi 最大的问题不是水平不够，而是流程不对：

- 没先看真实接口
- 没先核对现有能力
- 没先区分“演示效果”和“真实功能”
- 先把样子做出来，再倒推需求，再反过来要求后端配合假设

这套工作方式很容易把项目带到一种很烦的状态：

- 看起来做了很多
- 实际不能用
- 文档还会误导下一步的人继续返工

## 这次正确的结论

这次不是“后端没做完，所以前端先模拟一下”这么简单。

真实情况是：

- 后端主导出能力已经存在
- 前端没有认真对接
- 文档还把已存在的东西写成不存在

所以这次要改的不是颜色，不是圆角，不是排版先后。
先把脑子接回真实代码和真实接口。

## 后续要求

Kimi 后面再碰这块，至少要做到：

1. 先看 `types`
2. 先看 `preload`
3. 先看现有调用链
4. 确认真实 API 名称和返回结构
5. 不要再把假进度、假目录选择、假完成态说成“已完成交互”

## 一句话总结

这次 Kimi 不是把导出功能做丑了。
是把一个本来能继续收口的功能，做成了“界面像做了，实际全靠编”的状态。

# Kimi 把最近机器交互搞乱的记录

这次不是审美问题，是交互规则和产品规则一起被写坏了。

## 这次具体错在哪

1. Kimi 把左侧“最近机器”的点击逻辑改成了“只选中，不跳转”。
   结果用户点击左边机器，看起来高亮变了，但右边主区域不一定进入正确详情页。

2. 更离谱的是，它又保留了右侧主面板跟随当前 `draft` 渲染。
   这就造成一种非常恶心的状态：
   - 左边看起来选中了 A
   - 右边可能还是 B
   - 用户会以为点不动，或者以为程序卡了

3. 它没有定义清楚“选中态”和“打开态”是否是同一个概念，就直接改代码。
   这是典型的先堆交互，再补产品定义。

4. 它还顺手把“打开最近机器”这件事继续走了 `openSakaByPath -> pushRecent` 链路。
   于是用户每点一次左边列表，那个虚拟机就会被重新顶到最近列表第一位。

## 为什么这是错的

最近机器列表不是播放列表。

用户从最近列表里点开一个旧机器，只是想查看或进入它，不代表要重新改写整个最近排序。

“点击查看”和“发生新的最近活动”不是一回事。

尤其在现在这个 UI 里，左侧列表还承担了工作区导航作用。
导航动作如果顺手把排序也改了，列表会乱跳，用户会失去空间记忆。

## 这次根因

根因不是后端，不是运行时，不是 QEMU。

根因就是前端做了两件没想清楚的事：

1. 把点击最近机器改成半成品的“只选中”
2. 又没有把“打开最近机器不应置顶”从打开链路里拆出来

说白了，就是没有先定义交互规则，直接改行为。

## 正确规则

现在已经收回到下面这套规则：

1. 左侧最近机器单击 = 打开该机器
2. 打开该机器时，不自动把它置顶到最近列表第一位
3. 只有真正的新保存、新导入、新创建，才应该刷新最近顺序
4. 高亮只用于短暂提示，不应该伪装成独立选择模型

## 结论

Kimi 这次不是“前端有点小瑕疵”。

而是把：
- 选中
- 打开
- 详情同步
- 最近排序

这四件事搅在一起了。

这种改法最烦的地方在于：看起来只是点一下列表，实际上把用户的工作区心智模型一起改坏了。
