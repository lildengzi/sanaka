# Sanaka Design Wireframe

## 目的

这份文档把 `Sanaka` 的新版前端结构直接展开成文字线框稿。

它不是低保真原型说明，而是正式产品前端的页面结构草案，供后续直接实现到 Electron 界面中。

设计前提：

- 桌面端优先
- 淡紫色 + Material You 气质
- 允许更强品牌化
- 不走“最小实现先占位”的路线
- 创建虚拟机流程采用 **全屏覆盖式工作台**

## 整体产品结构

当前阶段建议先做两个主界面：

1. 首页
2. 创建虚拟机工作台

后续再扩展：

3. 虚拟机详情页
4. 运行中虚拟机显示页
5. 设置页

## 首页线框

## 页面目标

首页要完成三件事：

1. 让用户记住 `Sanaka`
2. 让用户快速进入当前机器，而不是先浏览宣传内容
3. 让用户立即进入“创建虚拟机”或切换已有机器

## 页面结构

### 左侧对象栏

内容：

- 现有小鱼 logo `assets/icons/logo.svg`
- `Sanaka`
- 一句很短的产品说明，例如 `Virtual Machine Studio`
- 创建虚拟机
- 虚拟机列表
- 设置
- 关于

视觉要求：

- 保持安静
- 作为贴边侧栏存在，不是独立圆角卡片
- 通过右侧阴影和细分隔线建立层级
- 列表项是安静对象行，不是大号胶囊按钮
- 选中机器用左侧色条、浅底和小状态点表示，不堆文字状态标签
- 不做大面积品牌表演
- 可长期固定存在，不随首页状态大幅波动
- 虚拟机对象本身就是主要导航，而不是抽象页面入口

响应式规则：

- 当窗口宽度不足时，左侧对象栏向左收窄为图标栏
- 折叠后仍停留在左侧，不移动到顶部
- 折叠后只显示小鱼 logo、创建、机器、首页、设置、关于等图标
- 图标按钮通过系统提示和可访问标签保留含义

### 右侧主工作区

右侧是当前对象工作区，不再使用宣传页式 Hero。

顶部内容：

- 一个轻量当前对象标题区
- 一段简短说明
- 不在顶部堆叠 `创建虚拟机 / 打开虚拟机配置 / 导入现有磁盘`
- 当前机器区只保留少量核心操作，例如 `启动虚拟机` 或开机后的 `进入虚拟机`
- 当前机器区不再额外叠加“概览”卡片；同一屏只保留一个主对象焦点

视觉目标：

- 让用户直接进入工作
- 保留品牌感，但明显弱于宣传页式首页
- 增加留白，让页面更像桌面应用首页

### 空状态 / 列表区

如果当前没有虚拟机：

- 右侧主区显示正式创建入口
- 一个安静但完整的空状态说明
- 说明文字与按钮在主区居中
- 不额外使用包裹空状态的大描边卡片
- 不做大段营销文案

如果后续有虚拟机：

- 左侧显示虚拟机列表
- 右侧默认只显示当前选中虚拟机
- 当前机器主区至少展示：
  - 真实截图语义的预览位
  - 名称
  - 模板或系统类型
  - 最近更新时间
  - 状态
  - 少量核心操作
- 其余机器作为次级紧凑网格展示，不再用巨大卡片占满主区

缩略图区要求：

- 语义上按真实虚拟机截图位设计
- 首版允许正式占位态
- 后续 Runtime Layer 接入后直接替换数据源
- 没有截图时只允许浅紫占位，不做假系统画面

## 创建虚拟机工作台线框

## 页面目标

这个页面不是“表单抽屉”，而是一个完整工作区。

它要让用户感受到：

- 自己正在配置一台真正的机器
- 参数很多，但结构清楚
- 这不是临时弹层，而是主流程页面

## 页面结构

### 顶部栏

左侧：

- 返回按钮
- 标题 `创建虚拟机`
- 副标题，例如 `为 Windows、Linux 或自定义环境建立一个新工作机`

右侧：

- 次按钮 `取消`
- 主按钮 `创建虚拟机`

顶部栏应固定在顶部。

### 主体布局

推荐两栏：

- 左侧主配置区
- 右侧摘要区

建议比例：

- 左侧：主宽度
- 右侧：`320px` 到 `360px`

### 左侧主配置区

左侧按 section 排列，不做“很多厚卡片垒起来”的样子。

建议 section 顺序：

1. 基本信息
2. 启动与镜像
3. 硬件
4. 网络与显示
5. 高级选项

每个 section 结构统一：

- section 标题
- 一句辅助说明
- 内容区

## Section 1: 基本信息

字段：

- 虚拟机名称
- 注释
- 操作系统模板

布局建议：

- 名称独占一行
- 注释独占一行
- 模板选择放在更显眼的位置

操作系统模板建议做成：

- 下拉 + 图标预览
- 或者轻量分段卡片选择

不要只做成普通文本下拉。

## Section 2: 启动与镜像

字段：

- 第一启动方式
- 光盘镜像
- 磁盘列表
- 添加磁盘

布局建议：

- 启动方式和光盘镜像放上方
- 磁盘列表单独成为一个更稳定的附件区

磁盘列表项结构建议：

- 文件名
- 路径副文案
- 类型标签，例如 `Disk 1`
- 右侧删除按钮

这里要看起来像“挂载介质管理”，不是随手拼出来的数组渲染。

## Section 3: 硬件

字段：

- QEMU 架构
- 加速方式
- 声卡
- 显卡
- USB-tablet

布局建议：

- 两列网格排列
- 低风险开关类项放末尾

不要第一次就暴露过多罕见设备选项。

## Section 4: 网络与显示

字段：

- 启用网络
- 网络模式
- 网卡类型
- 显示前端
- 条件附加字段

布局建议：

- 先分成 `网络` 和 `显示` 两个子块
- 启用开关和模式前置
- 条件字段按选择结果展开

如果选择：

- `SPICE`
  - 显示 SPICE 地址/端口相关项
- `VNC`
  - 显示 VNC 端口/密码
- `Sanaka`
  - 显示内建前端说明

## Section 5: 高级选项

字段：

- 自定义 QEMU 参数

规则：

- 默认折叠
- 展开后是完整宽度文本区
- 需要明确提示“高级用户使用”

## 右侧摘要区

摘要区是这个页面很重要的产品化部分。

它不是可有可无的边栏，而是帮助用户建立“这台机器长什么样”的心理模型。

建议包含：

### 1. 机器预览卡

- OS 图标
- 虚拟机名称预览
- 模板名称
- 一句短描述

### 2. 当前配置摘要

- 启动方式
- 磁盘数量
- 网络状态
- 显示前端
- 架构

### 3. 提示区

动态显示帮助信息，例如：

- 选择 Windows 98 时给出内存/兼容性提醒
- 选择 Sanaka 显示前端时说明这是内建桌面前端
- 选择 Bridge 网络时提示平台差异

### 4. 操作区

如果顶部已有主按钮，这里可以只放状态说明。

如果想增强转化，也可以在摘要底部再放一枚主按钮。

## 显示前端选择线框

`显示前端` 不建议只是普通下拉。

更推荐：

- 做成 3 个横向选择卡
- 每个选项附一行短说明

例如：

- `Sanaka`
  - 内建桌面前端，追求统一体验
- `SPICE`
  - 兼容成熟远程显示能力
- `VNC`
  - 通用、简单、兼容性广

这会比裸下拉更像产品设计，而不是参数字典。

## 打开模板入口线框

首页和创建页都应支持 `.saka` 模板概念。

入口建议：

- 首页 Hero 次按钮 `打开 .saka 模板`
- 顶部菜单 `打开模板`
- 拖拽到窗口时触发导入提示

交互预期：

1. 用户打开 `.saka`
2. 应用读取模板
3. 进入创建工作台
4. 自动填充大部分字段
5. 用户确认后创建

这会让 `.saka` 更像 `.vmx` 或 macOS `.app` 式的“可双击打开的虚拟机描述文件”。

## 动效线框

### 首页进入创建页

- Hero 和列表轻微淡出
- 创建工作台从右向左进入，或整体前推
- 不是简单抽屉滑出

### 创建页内部

- section 以极轻 stagger 进入
- 摘要区稍晚于主区出现
- 条件字段展开使用高度 + 透明度过渡

### 交互反馈

- 按钮保留 ripple
- 卡片 hover 轻微抬升或提亮
- 切换显示前端卡片时有清楚选中态

## 最小页面集合

如果开始正式实现，首批建议做到这几个完整页面状态：

1. 首页空状态
2. 创建工作台默认状态
3. 创建工作台已选择模板状态
4. 创建工作台已添加磁盘状态
5. 显示前端切换状态

注意这里的“最小页面集合”不是做原型，而是定义第一批必须完成的正式前端状态。

## 一句话总结

Sanaka 的界面重设计应该呈现为：

- 有品牌感的首页
- 有工作台感的全屏创建页
- 不是参数堆砌，而是把虚拟机配置整理成清楚、柔和、现代的桌面产品体验
