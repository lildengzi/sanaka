# Sanaka Design Spec

## 目标

这份设计规范用于指导 `Sanaka` 当前 Electron 前端的重设计。

目标不是做成通用后台，也不是做成移动端设置页，而是做成一个：

- 桌面端优先
- 带有 Material You 气质
- 使用淡紫色作为主品牌色
- 动效柔和
- 信息层级清晰
- 适合“创建和管理虚拟机”的轻量桌面应用

当前阶段只考虑前端，不考虑后端和真实 QEMU 执行。

但这里有一个明确约束：

- 不做“演示原型式前端”
- 不做“看起来像能用的最小实现”
- 不做“先搭个壳子再说”的页面

即使暂时没有后端，前端也必须按 **正式产品界面** 来设计和实现。

这意味着：

- 视觉层级必须完整
- 页面结构必须可持续扩展
- 组件规则必须统一
- 允许并鼓励更明确的品牌化表达

## 设计方向

### 关键词

- Calm
- Soft
- Technical
- Modern
- Desktop-first
- Material You inspired
- Branded

### 气质要求

界面应当同时满足三件事：

- 看起来柔和、轻盈、友好
- 仍然像一个专业工具，而不是消费类玩具
- 有明确的品牌存在感，而不是无印象的系统壳子

因此要避免：

- 过深的阴影
- 过强的饱和度
- 廉价感的花哨装饰
- 大面积纯白配一点点浅紫的空洞感
- 把所有内容都做成一样的圆角卡片
- 为了“最小实现”把界面做得过于朴素、无性格

### 产品级要求

Sanaka 的前端应按“正式产品界面”而不是“临时原型”处理。

具体要求：

- 首页要有明确视觉重心，但必须回到桌面工作台而不是大 Hero
- 创建流程要有明确页面结构
- 组件不允许临时拼装
- 页面要让人感到这是一个正在成形的产品，而不是一个演示稿
- 不允许再通过外层整块底板、超大标题、渐变按钮去强行制造存在感

允许更强的品牌化表达，例如：

- 更稳定的品牌识别区
- 带辨识度的标题区
- 更明确的品牌色重心
- 更像工作台的创建页结构

## 信息架构

所有主页面应共享同一套桌面应用外壳：

- 窗口背景先保留一整行明显的顶部拖动栏
- 拖动栏下方再进入左侧对象栏和右侧内容区
- 左侧对象栏保持一致
- 右侧内容区使用同一套留白规则与工作区块
- 不允许首页是侧栏结构、详情页却退回顶部胶囊导航这种两套界面语言并存的状态
- 不允许再用一个大圆角底板把侧栏和主区整体包起来
- 左侧栏是贴边应用结构，不能做成独立圆角卡片
- 左侧栏通过右侧阴影和细分隔线建立层级，不能靠外框、大圆角或整块浮层表达
- 左侧栏的机器列表应更像对象列表，而不是一组大按钮；选中态优先用左侧色条、浅色底和状态点表达
- 小窗口下左侧栏默认向左收窄为图标栏，只保留 logo、创建、机器、首页、设置、关于等图标；不把侧栏搬到顶部，也不改成汉堡菜单
- 折叠图标栏仍必须保留可访问名称和 hover/title 提示，不能让用户完全猜图标含义
- 右侧内容区每个视口只允许一个主焦点，不允许“当前机器大区 + 额外概览卡 + 重复摘要”同时抢层级

## 首页

首页应分为两种状态：

### 1. 空状态

当没有虚拟机时，首页展示：

- 左侧对象栏
- 右侧主工作区
- 右侧中的正式创建入口
- `打开虚拟机配置` 与 `导入现有磁盘` 次入口
- 一段简短说明，但不使用宣传页式大标题

右下角 FAB 可以保留，但不能成为唯一主入口。

空状态首页必须像正式产品首页，而不是“页面还没做完时的占位状态”。

品牌表达方式在首页应改为：

- 左上品牌区使用现有鱼 logo
- 留白与层级体现气质，而不是超大 Hero
- 用较轻的当前对象标题区或静态说明区建立重心
- 控制装饰元素数量，不让首页像宣传页
- 空状态主体采用居中文字与按钮，不再放截图占位图
- 空状态主体不额外套大型描边卡片，避免再次制造“内容太满”的感觉
- 右侧工作区整体宽度需要收窄并居中，不能横向撑满
- 顶部拖动栏与主体之间留出明显呼吸感

### 2. 有机器状态

后续如果有虚拟机，首页结构应为：

- 左侧对象栏直接展示虚拟机列表
- 右侧默认只展示当前选中机器
- 当前机器区固定包含：
  - 预览区
  - 标题与状态
  - 关键信息
- 当前机器区只保留少量核心动作，不在标题区堆叠通用动作按钮
- 当前机器区不重复展示完整路径、内部文件名或同一组摘要；首页只保留用户判断当前对象所需的信息
- 其余机器优先通过左侧对象列表切换，不在首页默认追加解释性卡片区

当前阶段即使没有列表逻辑，页面结构也应为未来列表状态留位置。

### 首页额外约束

- 不再把首页首屏做成营销 Hero
- 不再使用“Build a machine, not a checklist.” 这类宣传口号作为首页主视觉
- 首页文案不主动暴露 `.saka` 文件格式名
- 首页主体不重复放大品牌 logo；logo 只在品牌区和 About 等合适位置强化
- 机器卡片必须保留“未来真实运行缩略图”的正式槽位
- 在尚未接入 Runtime Layer 时，缩略图区使用正式占位态，而不是随意假图或临时原型画面
- 机器对象列表必须优先于“首页 / 模板 / 设置”这类站点式导航
- 不允许再出现外层整块淡紫色底板
- 不允许把左侧栏做成独立圆角卡片
- 不允许标题过大、按钮过重、粗体过密
- 不允许在已有机器状态的首页顶部放一排通用动作按钮
- 不允许主按钮使用渐变
- 不允许把页面内容铺到接近窗口边缘
- 不允许把左侧对象栏做成浮动圆角卡；层级来自贴边侧栏的右侧阴影和分隔线
- 不允许把弱标签、状态、导航全部做成高存在感胶囊

## 新建虚拟机面板

不建议继续使用“无限加长的窄抽屉”。

建议改为 **全屏覆盖式创建面板**。

它的行为应当是：

- 从当前首页进入
- 覆盖几乎整个应用窗口
- 保留进入和退出动效
- 在体验上像一个专注的创建工作区，而不是临时侧边栏

这个面板本质上是一个“创建虚拟机工作台”。

它必须看起来像正式产品中的核心工作区，而不是临时弹出的表单集合。

### 面板结构

顶部区域应包含：

- 返回或关闭按钮
- 标题 `创建虚拟机`
- 一句简短辅助说明
- 右上角主操作按钮预留位

主体区域建议采用：

- 左侧主配置列
- 右侧摘要列

如果当前阶段不做摘要列，也至少要保留将来可扩展空间。

不建议继续把所有内容放进单一窄列里。

即使首版做单栏，也不能退回“最小可用原型”的思路。

要求仍然是：

- 页面有明确头部
- 区块有稳定层级
- 操作区有固定位置
- 品牌表达存在

### 分组顺序

1. 基本信息
2. 启动与镜像
3. 硬件
4. 网络与显示
5. 高级选项

### 每组内容

#### 1. 基本信息

- 虚拟机名称
- 注释
- 操作系统模板

这是用户最先理解的一组，不应被技术参数打断。

#### 2. 启动与镜像

- 第一启动方式
- 光盘镜像
- 磁盘列表
- 添加磁盘

这组表达“这台虚拟机从哪里启动、挂了哪些介质”。

#### 3. 硬件

- QEMU 架构
- 加速方式
- 声卡
- 显卡
- 外设开关

这组只保留高频项，不铺满全部可选硬件。

#### 4. 网络与显示

- 启用网络
- 网络模式
- 网卡类型
- 显示前端
- 条件附加字段

这组应当有明显条件显示逻辑，不要默认全部展开。

#### 5. 高级选项

- 自定义 QEMU 参数

默认折叠。只有需要的人才展开。

## 布局规范

## 窗口

- 最小设计宽度按桌面端 `1280px` 考虑
- 创建面板应占据 `92%` 到 `100%` 的窗口宽度
- 创建面板应占据整个窗口高度
- 主内容区域最大宽度建议 `1280px`

## 页面边距

- 页面主边距：`32px`
- 大屏可提升到 `40px`
- 卡片内边距：`20px` 或 `24px`
- 顶部拖动栏下沿到页面主体的起始留白建议不小于 `20px`
- 侧栏与主工作区之间建议保留 `24px` 到 `32px` 间隔
- 首页主工作区建议控制在 `880px` 到 `1040px` 的视觉宽度内，不要横向铺满
- 首页的次级机器集合建议默认使用 `2` 到 `3` 列，不允许单台机器宽卡撑满主区

## 栅格与间距

使用 4px 基线，但主要采用这些间距：

- `4`
- `8`
- `12`
- `16`
- `20`
- `24`
- `32`

不要出现太多随机值。

## 圆角

- 超大容器卡片：`24px`
- 普通卡片：`24px`
- 输入框/按钮：`18px`
- 小标签/胶囊：`9999px`

不要所有元素都使用同一个圆角值。

## 色彩规范

## 品牌主色

主方向必须是淡紫色，但不是单一紫色。

建议色板：

- `--md-sys-color-primary: #7c5cff`
- `--md-sys-color-on-primary: #ffffff`
- `--md-sys-color-primary-container: #ebe3ff`
- `--md-sys-color-on-primary-container: #25124d`

## 中性色

- `--md-sys-color-surface: #fcfbff`
- `--md-sys-color-surface-dim: #f3f0f8`
- `--md-sys-color-surface-container: #f2edf7`
- `--md-sys-color-surface-container-high: #ebe5f1`
- `--md-sys-color-on-surface: #1d1b20`
- `--md-sys-color-on-surface-variant: #645f6b`
- `--md-sys-color-outline: #c8c1cf`
- `--md-sys-color-outline-variant: #e1dbe7`

## 功能色

- 成功：偏冷绿，不要荧光绿
- 警告：柔和琥珀色
- 错误：低饱和红

这些颜色出现频率必须低于主色和中性色。

## 品牌化用色

允许在这些位置更明确地使用品牌色：

- 首页品牌区
- 创建面板标题区
- 关键主按钮
- 选中态和当前步骤强调

但品牌化不等于滥用纯紫背景。

更推荐：

- 纯色淡紫与 `primary-container` 的层次组合
- `primary` 与 `primary-container` 的层次组合
- 在中性色背景上建立品牌重心

## 用色规则

- 大面积背景使用 `surface`
- 可区分层级的模块使用 `surface-container`
- 主强调按钮使用 `primary`
- 低强度可点击区域优先使用 `primary-container`
- 分割线与输入边框使用 `outline-variant`

避免：

- 整页铺满紫色
- 只有按钮是紫色，其余全白导致界面发空
- 过多描边

## 字体规范

当前项目已有 `Montserrat`，可以继续沿用，但要控制使用方式。

### 字号层级

- 页面主标题：`22px / 28px`
- 区块标题：`18px / 22px`
- 输入标签：`13px / 16px`
- 正文：`14px / 20px`
- 辅助说明：`12px / 18px`
- 按钮文字：`14px / 14px`

### 字重

- 页面主标题：`600`
- 区块标题：`600`
- 正文：`400` 或 `500`
- 说明文字：`400`
- 导航、按钮、副标题、状态文本默认使用 `400` 或 `500`
- 粗体只保留给品牌名、页面主标题、卡片主标题

不要让整个页面都用粗体，也不要再出现超大粗标题作为首页主视觉。

## 组件规范

## App Bar

顶部栏应为轻量桌面栏：

- 高度 `64px`
- 左侧应用名或留白拖动区域
- 中间不放复杂导航
- 右侧只保留少量应用级操作

顶部栏背景不应纯透明，建议使用带轻微模糊或浅层 surface。

它不必完全中性，也可以带有轻微品牌识别，例如：

- 更有辨识度的标题排印
- 左上角品牌标识
- 极轻量的品牌底色倾向

## 按钮

### 主按钮

用于：

- 创建虚拟机
- 保存
- 继续

规则：

- 使用 `primary` 背景
- 文本白色
- 圆角 `18px`
- 高度 `44px` 到 `48px`
- 禁止渐变、发光和夸张高对比装饰

### 次按钮

用于：

- 取消
- 导入
- 选择文件

规则：

- 使用 `primary-container` 或 `surface-container-high`
- 不要默认描边风格太重
- 字重保持常规，不抢主按钮层级

### 图标按钮

- 默认安静
- hover 时只做轻微底色浮现
- 不使用实体色圆底，除非它是特殊工具按钮

## FAB

FAB 可以保留，但要弱化“唯一主入口”的角色。

规则：

- 尺寸 `56px`
- 使用 `primary-container`
- 带 ripple
- 悬浮阴影非常轻

## 输入框

输入框建议统一使用一种风格，不要混搭。

推荐：

- 背景使用 `surface-container`
- 边框用 `outline-variant`
- focus 时边框切到 `primary`
- 文本左对齐
- 标签放在输入框上方，不做浮动 label

## 下拉菜单

下拉需统一成同一种组件模型：

- 默认关闭态看起来像输入框
- 展开菜单带圆角和轻微浮层
- 选中态使用 `primary-container`
- hover 不要太深

长列表必须限制最大高度并可滚动。

## 卡片

卡片不是为了“把每一段都框起来”，而是为了制造层级。

规则：

- 同一页面最多 2 到 3 种卡片强度
- 首页空状态卡片可以更大、更柔和
- 抽屉里的分组卡片应该比页面背景只高一层，不要像浮在空中

卡片不是唯一的品牌载体，品牌感更应来自：

- 页面结构
- 色彩重心
- 标题区
- 空状态视觉
- 动效质感

## Switch / Checkbox

启用型开关应当更像真正的 on/off 控件，而不是普通复选框。

建议：

- 网络开关
- USB-tablet 开关

都使用 Material 风格轨道开关。

## 磁盘列表

磁盘区域应从“临时列表盒子”改成更明确的附件列表。

单项结构建议：

- 文件名
- 可选的路径副文案
- 右侧删除按钮

列表项不应过高，也不应做成大色块按钮感。

## 文案规范

文案要像桌面产品，不像技术文档。

建议：

- 少用括号解释一大串
- 主文案短
- 说明文案弱化
- 专业参数放在辅助说明，而不是标题里

例如：

- `操作系统模板`
- `启动介质`
- `网络模式`

比：

- `操作系统（模版）`
- `光盘镜像（文件选择）`
- `虚拟机加速设置`

更自然。

## 创建面板规范

## 进入方式

点击首页主按钮或 FAB 后，进入全屏覆盖式创建面板。

这个面板应：

- 视觉上独立于首页
- 进入后让用户明确感知自己已进入一个新的任务上下文
- 不再让首页内容继续争夺注意力

## 顶部栏

创建面板顶部栏建议包含：

- 左侧返回按钮
- 主标题
- 副标题
- 右侧主操作按钮，如 `创建`

标题区要明显强于普通卡片标题，像页面标题，不像表单小节标题。

## 主体布局

推荐两栏：

- 左侧 `minmax(0, 1fr)` 主配置区
- 右侧 `320px` 到 `360px` 摘要区

右侧摘要区可用于：

- 显示操作系统图标
- 显示虚拟机名称预览
- 显示已选磁盘数量
- 显示启动方式和网络状态
- 放置辅助说明或模板提示

如果首版实现压力大，也可以先做单栏，但视觉结构仍要按“页面”而不是“抽屉”来设计。

## 区块规则

每个大分组应是一个独立 section，但不要做成一排厚重卡片。

建议：

- section 之间用留白区隔
- 卡片只提升一层
- 标题、说明、内容形成稳定节奏

不要让创建面板看起来像一个长设置页截图。

## 设置页边界

- 设置页只承载应用级偏好，不承载每台虚拟机应自行决定的运行细节
- 运行默认项只保留应用级显示默认值
- 不在设置页提供默认网络模式；网络模式属于单台虚拟机配置，创建页默认 `User`
- 不在设置页提供默认音频方式；音频方式移动到单台虚拟机高级选项，默认 `系统自动`
- 不让普通用户在应用设置页被迫理解 PipeWire、PulseAudio、CoreAudio、DirectSound 等平台音频后端

## 页脚操作区

底部建议有固定操作区，至少包含：

- `取消`
- `创建虚拟机`

如果使用顶部主按钮，则底部可以只保留次操作，但仍应保证长页面滚动时操作不会消失得太彻底。

## 动效规范

## 总原则

动效必须服务于层级变化和状态变化，不为了炫。

要求：

- 短
- 柔和
- 有方向感
- 不拖泥带水

同时要求：

- 动效要增强品牌感
- 动效不能只是默认 UI 过渡

Sanaka 的动效应当给人一种：

- 温和
- 精致
- 有设备感
- 带一点未来桌面工具气质

## 建议时长

- hover：`120ms` 到 `160ms`
- 普通展开/收起：`180ms` 到 `240ms`
- 抽屉进入：`240ms` 到 `320ms`
- 页面级内容淡入：`300ms` 左右

## 建议曲线

- 标准过渡：`cubic-bezier(0.2, 0, 0, 1)`
- 进入动画：`cubic-bezier(0.05, 0.7, 0.1, 1)`

## 具体动效

### About 卡片

- 淡入
- 轻微上移/下落
- 不要弹跳

### 创建面板

- 从右向左进入，或轻微放大淡入
- 同时伴随透明度变化
- 不使用硬切
- 动效结束后，用户应明显感觉自己已进入“创建模式”

首页与创建面板之间应存在明确的层级切换，而不是像打开一个普通设置栏。

### 区块内容

- 打开创建面板后，各表单组可做很轻微 stagger
- 每组延迟应非常小，避免做成演示稿效果

### 按钮反馈

- hover 只轻微提亮或加深
- active 略微压下
- 保留 ripple，但范围要克制

## 可用性规则

## 默认优先级

默认状态下，用户第一眼应该看到：

1. 这是什么应用
2. 如何创建虚拟机
3. 进入创建后先填什么

不是先看到一堆硬件名词。

## 降低技术压迫感

虽然这是 QEMU 管理器，但首次界面不应压用户一脸参数。

因此：

- 高频项前置
- 低频项折叠
- 长列表限制暴露范围
- 复杂解释放到 note

## 空状态设计

空状态不能太空。

需要：

- 清楚的标题
- 一句说明
- 主行动按钮
- 视觉锚点图形

## 当前阶段的实现优先级

如果开始重做前端，优先级建议如下：

1. 重写首页空状态
2. 重写创建面板结构
3. 重写颜色 token 和间距系统
4. 统一输入、按钮、下拉组件
5. 重做磁盘列表样式
6. 最后再调整 about 卡片和细节动画

## 不做的事

当前阶段不建议做这些：

- 追求完整 Figma 级多页面系统
- 一次性覆盖全部 QEMU 参数
- 拟物或重阴影风格
- 把桌面应用做成手机界面放大版
- 用“最小实现”思路把正式界面做成低完成度草稿
- 仅仅为了占位而写原型式页面

可以做的事：

- 品牌化
- 更完整的首页视觉
- 更强的创建工作台感
- 更有辨识度的标题区和空状态

## 动画与全屏过渡规范 (Transition & Animation Guidelines)

为了提升应用的物理真实感和空间连贯性，Sanaka 的关键页面切换（开机、进入控制台、删除配置）不应使用普通的淡入淡出或瞬切，而是使用**非线性全屏膨胀过渡**：

- **图标膨胀动画**：
  - 点击“启动”或“进入控制台”时，对应的操作图标（播放三角形/显示器/垃圾桶）会以非线性方式（使用 `cubic-bezier(0.7, 0, 0.3, 1)`）快速放大至 150 倍。
  - 在放大至 45% 进度时（450ms），图标的实体色应完全覆盖整个视口，此时在后台完成路由切换或数据写入。
  - 从 50% 进度开始，覆盖色平滑渐变到目标页面的背景色（`var(--bg)`），并在后半段（550ms-1000ms）柔和淡出，露出已经加载完毕的新页面。
- **内置模态弹窗**：
  - 严禁使用任何操作系统原生的 `window.confirm` 或 `window.alert` 等阻塞进程的对话框，它们破坏了应用的整体设计语言。
  - 使用内置的磨砂半透明背景层 `.modal-backdrop`（具备 `backdrop-filter: blur(8px)`）和卡片容器 `.modal-card`。
  - 弹出时配合轻微的弹性放大动效（`scale(0.985) -> scale(1)`），确认后通过触发全屏膨胀动画进行转场。

## 一句话总结

`Sanaka` 的重设计应当是：

**淡紫色、柔和、现代、桌面优先、带 Material You 气质、并且有明确品牌感的正式产品前端。**

不是纯配置页，不是演示原型，也不是花哨概念稿，而是一个可以直接落成 Electron 正式界面的可执行设计规范。
