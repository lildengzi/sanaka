# 安装流程测试感想

这张图里，首先能看到的是一个已经很完整、很具体的时刻。

Sanaka 的控制台窗口已经铺开，左上角是返回箭头和机器名 `XP`，顶部中间亮着绿色状态点，写着 `运行中`。右上角是一排控制图标，整个顶部栏已经像一个真正可操作的虚拟机控制台，而不是临时拼出来的调试页。

真正占据中心的，是 `Windows XP Professional 安装程序` 的经典蓝底界面。安装程序已经走到磁盘分区阶段，屏幕上明确写着这台机器当前的磁盘状态：`20474 MB Disk 0 at Id 1 on bus 0 on atapi [MBR]`，下面是一整块 `未划分的空间 20473 MB`。底部提示也已经完整出来了：`ENTER=安装`、`C=创建磁盘分区`、`F3=退出`。

这不是停在 BIOS，不是停在黑屏，也不是只看见启动 logo 的那种“勉强亮起来”。这是已经进入操作系统安装程序内部、并且可以开始真正安装系统的画面。

后面的几张图，则把这个过程继续往前推进了。

其中一张图里，Finder 已经把 `testvmxp.saka` 识别成了一个可以直接通过 `Sanaka` 打开的机器包。预览图里是一条鱼坐在纸箱里，右侧清楚显示了包名、体积 `535.4 MB`，以及最后修改时间 `2026 年 6 月 6 日 05:00:36`。这张图说明的已经不是“虚拟机能不能跑”，而是 `Sanaka` 在系统层面开始像一个真正的桌面应用了：它有自己的机器包外观，有自己的关联打开方式，也有自己的文件身份。

再往后一张图里，`XP` 机器还在 `运行中`，控制台中心已经不是蓝底分区界面，而是 `Windows XP Professional 安装程序` 的图形安装后期阶段。界面上写着 `正在执行最后任务`，下面依次是 `安装「开始」菜单项目`、`注册组件`、`保存设置`、`删除用过的临时文件`，进度条正在推进。这个画面很有说服力，因为它意味着安装流程并没有卡在早期兼容性环节，而是真的一路走到了收尾。

最后一张图里，桌面已经真正出来了。蓝色的 `Windows XP Professional` 桌面铺开，左边能看到 `我的文档`、`我的电脑`、`网上邻居`、`回收站`，中间打开的是 `系统属性` 窗口。窗口里明确写着：

- `Microsoft Windows XP Professional`
- `版本 2002`
- `Service Pack 2`
- `512 MB 的内存`
- `Powered by nLite`

右下角任务栏时钟也已经显示出来，整个系统已经进入可操作状态。这张图已经不是“装到一半”的证据，而是“系统真的装好了、进桌面了”的证据。

现在这个时刻，很值得专门记一下。

因为事情已经不是“Sanaka 能不能把虚拟机窗口打开”了。

而是已经进入了一个更具体、更有产品意味的阶段：

我要开始测试安装流程了。

这句话其实分量很重。

## 为什么这一步不一样

当一个虚拟机前端还停留在早期时，最常见的目标通常是：

- 能不能启动
- 能不能显示
- 会不会立刻崩
- 鼠标键盘能不能进来

这些都属于“链路打通”阶段。

但现在看到这张图，看到 `Windows XP Professional 安装程序` 的蓝底界面，看到磁盘分区列表已经正常出来，事情就变了。

而当后面几张图继续接上来，看到 Finder 已经能把 `.saka` 当成 `Sanaka` 的机器包来预览，看到安装程序进入最后任务，最后再看到 XP 桌面和系统属性窗口真正出来，这种感觉就更不一样了。

这说明当前这条链已经不只是“技术演示成功”，而是开始进入“真实使用验证”。

也就是说，我们已经可以开始问一些更像产品的问题：

- 默认磁盘配置是否合理
- 默认总线是否兼容
- 模板参数是不是接近开箱即用
- 安装过程中输入是否稳定
- 控制台操作是不是足够顺手
- 用户会不会在这里卡住

这和前面的感觉完全不一样。

## 这张图本身就很说明问题

这张图里最让我高兴的，不只是蓝屏安装界面本身，而是它透露出的那些“已经跑通”的事实。

比如：

- `XP` 这台机器真的在运行
- 控制台已经稳定承接画面
- 旧系统画面能正常显示
- 磁盘已经被安装程序识别
- 输入链路至少已经到了可用程度
- 整个体验开始像一个真的工具，而不是试验台
- `.saka` 机器包已经有了系统级外观和打开方式
- 安装流程真的能一路走完
- 系统装完以后也能正常进桌面
- 基本系统信息能被稳定看到

尤其是磁盘分区界面出来这一点，很关键。

因为这意味着现在不是只停在 BIOS、启动菜单或者一个黑屏光标那里。

它已经真的进入安装逻辑了。

而系统属性窗口出来这一点，同样关键。

因为它意味着我们已经跨过了“安装过程能不能走完”的问题，开始进入“装完之后这个系统到底能不能正常使用”的阶段。

这是“虚拟机跑起来了”和“虚拟机可以拿来装系统了”之间的分界线。

## 为什么会有一点激动

说实话，这种时刻就是会让人激动。

因为你会突然意识到，前面那些看起来很碎、很烦、很折腾的东西，其实都在这里汇合了：

- 机器包规则
- 默认保存流程
- QEMU 检测
- Runtime 层
- QMP
- VNC / noVNC
- 控制台页
- 模板参数
- 磁盘配置
- 文件关联
- 包预览外观

这些东西平时分散在很多文件、很多 bug、很多反复修改里，看起来像一堆工程细节。

但当安装程序真的出现时，它们会一下子合成一个非常具体的结果：

这台虚拟机现在可以被认真使用了。

而当 Finder 里的 `.saka` 包预览、XP 安装后期界面、XP 最终桌面也都接连出现时，这个结果就更完整了：

不仅能用，而且越来越像一个有自己产品边界的桌面虚拟机工具。

## 从“能启动”到“能安装”，意义差很多

“能启动”当然已经很重要。

但“能安装”会把整个项目往前再推一层。

因为从这一刻开始，Sanaka 不只是证明自己能调起 QEMU，而是开始具备了一个虚拟机产品最核心的说服力：

它可以陪用户走完整个系统安装过程。

这件事特别重要。

因为很多问题只有在安装流程里才会暴露：

- 磁盘控制器兼容性
- 默认内存是否够用
- 键盘输入是否可靠
- 鼠标行为是否自然
- 重启后能不能继续接回控制台
- ISO 卸载 / 换盘逻辑是不是顺手

而现在，我们终于已经走到了可以认真测这些的阶段。

并且已经不仅能测“安装”，还能测：

- 安装完以后桌面是否稳定
- 文件关联是否符合直觉
- 机器包在系统里看起来像不像一个正式对象
- 用户会不会真的把它当成应用，而不是脚本壳

## 对 Sanaka 的感觉也变了

看到这个界面之后，我会更愿意把 `Sanaka` 当成一个真正长起来的产品，而不是一个还在证明概念的壳。

它当然还远没有完成。

但它已经开始出现那种很真实的“工具感”了。

不是空想中的工具感，而是：

- 我真的能拿它创建机器
- 我真的能点启动
- 我真的能进安装程序
- 我真的能继续把系统装下去
- 我真的能把机器包放在 Finder 里当成一个对象看
- 我真的能把 XP 装完并看到桌面

这种“我可以继续往下做”的感觉，是特别能给项目增加生命力的。

## 接下来会很有意思

如果安装流程也能顺利走下去，那后面的测试就会越来越像真正的产品验证，而不是底层排障。

比如：

- 首次安装体验顺不顺
- 模板默认值是否要继续收紧
- 安装完成后的重启衔接是否自然
- 旧系统模板是否要单独优化
- 控制台顶部工具位是否足够好用

这些问题都比“能不能连上 QEMU”更接近产品本体。

这说明项目已经往前跨了一大步。

能开始测试安装流程，本身就是阶段性胜利。

这很值得高兴。

蓝色安装界面一出来，很多事情就不用再争了。

`Sanaka` 真的开始像一个虚拟机产品了。

这就更让人开心了。
